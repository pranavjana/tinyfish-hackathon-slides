02 / 09
THE IDEA

What is
Web Agent?

A web automation API by TinyFish. Give it a URL and a goal in plain English — it drives a real browser and returns structured JSON.

URL + Goal
→
Real browser
clicks, types, scrolls
→
JSON result
  • No CSS selectors, no XPath — just natural language goals
  • Works on JS-heavy, authenticated, and bot-protected sites
  • SDKs for Python (tinyfish) and TypeScript (@tiny-fish/sdk)
03 / 09
STEP 1

Get Your
API Key.

1. Go to agent.tinyfish.ai/api-keys
2. Sign up / log in
3. Click "Create API Key"
4. Copy & store it — shown only once!

Keys start with sk-tinyfish-. Never commit them to version control.

# Add to your shell profile (.bashrc, .zshrc)
export TINYFISH_API_KEY="sk-tinyfish-*****"
04 / 09
THE API

Three
Endpoints.

/run
Synchronous. Send request, wait, get result. Simplest option.
Quick tasks under 30s
/run-async
Fire & check. Returns a run_id immediately. Poll for result later.
Batch processing, long tasks
/run-sse
Live stream. Real-time SSE events: STARTED, PROGRESS, COMPLETE.
User-facing apps, progress UI
POST https://agent.tinyfish.ai/v1/automation/{run | run-async | run-sse}
Header: X-API-Key: $TINYFISH_API_KEY
Body:   { "url": "...", "goal": "...", "browser_profile": "lite" }

06 / 09
PATTERN 01

Live Browser
Preview.

Every run emits a STREAMING_URL event — a live feed of the browser you can embed directly in your app. Users watch the automation happen in real time.

  • URL valid for 24 hours — works as a replay too
  • Read-only: users can watch, not interact
  • Keep your API key server-side — call the SSE endpoint from your backend
GET THE URL
// Listen for the SSE event
{ type: "STREAMING_URL",
  run_id: "abc123",
  streaming_url: "https://tf-abc123..." }
EMBED IT
<iframe
  src={streamingUrl}
  style="width:100%; aspect-ratio:16/9"
/>

08 / 09
PATTERN 03

Writing Good Goals.

BAD
"Get data"

"Extract everything"

"Click #btn-submit"

Vague goals, CSS selectors

GOOD
"1. Dismiss the cookie banner
 2. Wait 2 seconds
 3. Click the blue 'Add to Cart'
    button below the price
 4. Extract the cart total"

Numbered steps, visual descriptions, explicit waits

  • Describe elements visually — "the blue button below the price" beats #btn-submit
  • Number your steps for multi-step workflows
  • Add explicit waits at checkpoints (login pages, slow loads)
  • Include fallbacks: "If 'Access Denied' appears, return { error: 'blocked' }"
